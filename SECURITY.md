# 安全策略

## 支持范围

默认仅保证当前主分支上的最新代码得到修复与响应。若你在使用历史提交或长期未更新的分支，请先尝试复现到最新版本。

## 报告方式

- 不要在公开 Issue 中直接披露可利用的漏洞细节。
- 优先使用 GitHub Security Advisory 私下提交。
- 若仓库尚未启用私密安全通道，请先联系维护者建立非公开沟通，再决定披露方式。

## 报告内容

请尽量提供：

- 漏洞类型与影响范围
- 复现步骤或最小 PoC
- 触发条件、权限要求、前置配置
- 可能影响的数据、接口、部署方式
- 临时缓解方案（如果已有）

## 响应原则

- 维护者应先确认问题真实性与影响级别。
- 修复发布前，尽量避免公开可直接复现的攻击细节。
- 修复完成后，建议补充测试、变更说明和受影响版本范围。
